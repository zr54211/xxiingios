# BarcodeScannerZXing — внешняя компонента сканирования штрихкодов для мобильной платформы 1С

Кросс-платформенная (Android + iOS) внешняя компонента Native API на базе
декодера [zxing-cpp](https://github.com/zxing-cpp/zxing-cpp) (v3.1.0, Apache-2.0).

**Зачем:** штатный сканер мобильной платформы не читает плотные QR высокой версии
(мятая плёнка, блики, мелкие модули) — нет управления разрешением потока анализа,
фокусом и зумом. Intent-костыль со сторонним приложением работает только на Android.
Компонента даёт полный контроль над камерой и единый декодер на обеих ОС;
работает без Google-сервисов (важно для кассовых терминалов без GMS).

**Ограничение:** только собственные мобильные приложения, собранные сборщиком 1С
(в стандартный мобильный клиент ВК не подключается). Минимальная целевая
мобильная платформа — 8.3.27.

## Контракт (BSL)

```bsl
Компонента = Новый ("AddIn.<ИмяПодключения>.BarcodeScannerZXing");
```

| Член | Тип | Описание |
|---|---|---|
| `Версия` / `Version` | свойство, чтение | `"0.1.0 (zxing-cpp 3.1.0)"` |
| `НачатьСканирование(НастройкиJSON)` / `StartScanning` | процедура | Открыть экран сканера (Android и iOS); настройки `{"torch":false,"autoClose":true}` |
| `ЗавершитьСканирование()` / `StopScanning` | процедура | Закрыть экран сканера; в паузу автозакрытия после распознавания не обрывает её |
| `РаспознатьКадр(Данные, Ширина, Высота)` / `DecodeFrame` | функция → строка JSON | Распознать luminance-буфер (Y-плоскость, Ширина×Высота байт) |

Результат сканирования с экрана приходит через `ВнешнееСобытие`:
`Источник = "BarcodeScannerZXing"`, `Событие = "ScanResult"`, `Данные` — JSON:

```json
{"found":1,"barcodes":[{"format":"QR Code","text":"...","points":[[x,y],[x,y],[x,y],[x,y]]}]}
```

При автозакрытии событие отправляется **после** паузы со стоп-кадром (см. ниже) —
к моменту прихода `ScanResult` экран сканера уже закрывается.

## Поведение экрана сканера

- Читаются матричные (QR, DataMatrix, Aztec, PDF417…) и все линейные форматы
  (EAN/UPC, Code 128 и т.д.); линейные — включая наклонные, изогнутые (этикетка
  на бутылке) и повёрнутые на ~45° коды (дополнительный проход по повёрнутому кадру).
- Рамка из четырёх уголков сопровождает найденный код; для наклонного линейного кода
  ложится наклонным параллелограммом по реальному положению штрихов.
- При распознавании (в режиме автозакрытия): подача кадров останавливается, поверх
  превью показывается **цветной стоп-кадр того кадра, на котором код распознан**,
  уголки добегают до кода по застывшей картинке, пауза 0.4–0.5 с, затем `ScanResult`
  и закрытие. Событие после паузы — намеренно: обработка штрихкода в BSL занимает
  UI-поток и остановила бы анимацию рамки.
- Фонарик, tap-to-focus, кнопка ✕; «Назад» на Android закрывает сканер.

## Структура

```
core/                      — общий C++: Native API-слой + обвязка zxing-cpp
                             (восстановление высоты/наклона 1D, 45°-проход)
android/                   — NDK Camera2 + оверлей (java-мост UiBridge/ScannerOverlay)
ios/                       — AVFoundation + оверлей в собственном UIWindow (см. README)
vendor/1c-native-api/      — заголовки 1С Native API (см. README про сверку)
deps/zxing-cpp/            — сабмодуль, зафиксирован на теге v3.1.0
package/                   — manifest.xml + упаковка zip-макета (+ make-java.ps1)
```

## Сборка

Требования: CMake ≥ 3.21, C++20-компилятор; для Android — NDK r23+; для iOS — Xcode.

```powershell
# Windows (отладка контракта в тонком клиенте)
cmake -B build-win -DCMAKE_BUILD_TYPE=Release
cmake --build build-win --config Release

# Android / iOS — см. android/README.md и ios/README.md
# Упаковка zip-макета
pwsh package/make-zip.ps1
```

После клонирования: `git submodule update --init`.

## Состояние

Обе платформы готовы и проверены на устройствах: Android (arm64-v8a,
armeabi-v7a, x86_64 — полноэкранная Activity со сканером) и iOS (arm64,
статическая линковка, регистрация через `RegisterLibrary` — см. `ios/README.md`).
Финальный zip-макет содержит все архитектуры (`package/manifest.xml`).

Интеграция в МобильнуюКассу оформлена подсистемой **СканированиеZXing**
(ПоддержкаОборудования → ПодключаемоеОборудование): общий модуль
`СканированиеZXingКлиент` + общий макет `КомпонентаСканированияZXing`;
все вставки в типовые модули помечены комментариями
`// СканированиеZXing` … `// Конец СканированиеZXing`, инструкция встраивания —
в шапке модуля `СканированиеZXingКлиент`.
